## Backend Rules
- R-10: API base URL is always `https://api.janishammer.com` — never change without updating firmware
- R-11: Rate limiting uses D1-backed counter — NOT in-memory Map (multi-instance bug was fixed Apr 2026)
- R-12: PAYMENT_MODE = `fake` for all dev/test · `live` only when real ESP32 is physically connected
- R-13: Ghost devices = never use random MACs in test tools — every random MAC creates a new DB row
- R-14: Only 2 test devices allowed: `SATU-TEST001` (AA:BB:CC:DD:EE:00) + `SATU-SIM01` (AA:BB:CC:DD:EE:01)
- R-15: Run 14-test suite (satu-system-tester.html) after every backend change before closing

## Firmware Rules
- R-20: Variable declarations inside switch/case blocks MUST have braces `{}` — compile error otherwise
- R-21: device_id + device_secret must be persisted in NVS after /hello — never lost on reboot
- R-22: IR sensor E18-D80NK = NPN normally-open · SENSOR_TRIGGERED = LOW · mount 5-8cm below shelf
- R-23: Heartbeat every 5min · command poll every 30s · these are NOT negotiable timing values
- R-24: hardware.h abstraction supports both single and dual ESP32 — don't hardwire single-only logic

## Hardware Rules
- R-30: Relay board needs separate 12V supply — NOT from ESP32-S3 5V (max 500mA)
- R-31: Touch controller GT911 — may need separate library, NOT TFT_eSPI built-in
- R-32: Display driver EK9716 — confirm in TFT_eSPI User_Setup.h before flashing

## Security Rules

- R-41: All device auth uses device_id + device_secret pair — validate both on every call
